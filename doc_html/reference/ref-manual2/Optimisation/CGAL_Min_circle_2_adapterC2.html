Traits Class Adapter for 2D Smallest Enclosing Circle to 2D Cartesian Points (CGAL_Min_circle_2_adapterC2)

Definition

The class CGAL_Min_circle_2_adapterC2<PT,DA> interfaces the 2D optimisation algorithm for smallest enclosing circles with the point class PT. The data accessor DA [K{ü}hl 97] is used to access the x- and y-coordinate of PT, i.e. PT is supposed to have a Cartesian representation of its coordinates.

#include < CGAL/Min_circle_2_adapterC2.h>

Example

The following example illustrates the use of the traits class adapter with your own point class. For the sake of simplicity, we expect the point class to have double coordinates with access functions x() and y(). Based on this we show how to implement and use a data accessor for the point class.

The requirements catalogs of the point class and the data accessor are listed in Section .

Note: In this release correct results are only guaranteed if exact arithmetic is used, so this example (using inexact floating-point arithmetic) is only intended to illustrate the techniques. 

  #include <CGAL/Min_circle_2_adapterC2.h>
  #include <CGAL/Min_circle_2.h>

  /* your own point class */
  class Pt {
      ...
    public:
      ...
      Pt( double x, double y);
      ...
      double  x( ) const;
      double  y( ) const;
      ...
  };

  /* the data accessor for Pt */
  class Pt_DA {
    public:
      typedef  double  FT;
      void  get( Pt const& p, double& x, double& y) const
                { x = p.x(); y = p.y(); }
      double  get_x( Pt const& p) const { return( p.x()); }
      double  get_y( Pt const& p) const { return( p.y()); }
      void  set( Pt& p, double x, double y) const { p = Pt( x, y); }
  };

  /* some typedefs */
  typedef  CGAL_Min_circle_2_adapterC2< Pt, Pt_DA >  Adapter;
  typedef  CGAL_Min_circle_2< Adapter >              Min_circle;

  /* do something with Min_circle */
  Min_circle  mc(...);
