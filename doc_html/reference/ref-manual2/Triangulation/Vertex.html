Definition

A class must fulfill the following requirements in order that it can be used as a vertex of a triangulation.

The type Point should contain some geometric information such as coordinates or other, and is typically a point (for example Point could be CGAL_Point_2<R> for a triangulation of a planar point set, or CGAL_Point_3<R> for a terrain model). A vertex provides access to a face of the triangulation having v as vertex.

Types

Vertex::Point
The type of the geometric information associated to a vertex.

Vertex::Face
Face type.

Vertex::Vertex_circulator
A circulator for traversal of adjacent vertices.

Vertex::Face_circulator
A circulator for traversal of adjacent faces.

Creation



For triangulation algorithms designed by the user, vertices need to be explicitly constructed. If the modification of the triangulation is done through a triangulation class, there is no need for these functions.

Vertex v;
Introduces a variable v. The geometric information is initialized by the default constructor of class P. The pointer to the incident face is initialized with NULL.

Vertex v ( Point p);
Introduces a variable v, and initializes the geometric information. The pointer to the incident face is initialized with NULL.

Vertex v ( Point p, Face* f);
Introduces a variable v, and initializes the geometric information and the pointer to the incident face.



Setting



void v.set_point ( Point p) Sets the geometric information to p.
void v.set_face ( Face* f) Sets the incident face to f.



Access Functions

Point v.point () Returns the geometric information of v.
Face* v.face () Returns a face of the triangulation having v as vertex. The face is the first adjacent face in counterclockwise order, if v is on the boundary.

Miscellaneous Operations

bool v.is_on_boundary () Returns true if vertex v is on the boundary of the triangulation.
int v.degree () Returns the degree of v in the triangulation.

Traversal of the adjacent Vertices and Faces.

Two circulator classes allow to traverse the vertices and faces incident to a given vertex. The operator++ moves the circulator counterclockwise and operator-- moves the circulator clockwise. If the vertex is on the boundary the circulators perform a wrap-around at both ends of the adjacency list.

Vertex_circulator v.incident_vertices () Returns a circulator that allows to traverse the vertices incident to v starting at a vertex incident to v. If vertex v is on the boundary the circulator starts at the first vertex in counterclockwise order.
Face_circulator v.incident_faces ( Face* f = NULL)
Returns a circulator that allows to traverse the faces that are incident to v. The traversal starts at face f. If parameter f is omitted, the traversal starts at an arbitrary face incident to v, or at the first adjacent face in counterclockwise order, if v is on the boundary.

A face circulator is invalidated by any modification of the face it points to. A vertex circulator that turns around vertex v and that has as value a pointer to vertex w, is invalidated by any modification of the two faces that are incident to v and w.
