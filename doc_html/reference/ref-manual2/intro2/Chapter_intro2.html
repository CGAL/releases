Introduction

CGAL, the Computational Geometry Algorithms Library, is written in C++ and consists of three parts. The first part is the kernel, which consists of constant size non-modifiable geometric primitive objects and operations on these objects. The objects are parameterized by a representation class, which specifies the underlying number types used for calculations. The second part is a collection of basic geometric datastructures and algorithms, which are parameterized by traits classes that define the interface between the datastructure or algorithm, and the primitives they use. The third part consists of non-geometric support facilities, such as `circulators', random sources, I/O support for debugging and for interfacing CGAL to various visualization tools. Some of these facilities may depend on other tools or libraries, such as LEDA.

This part of the reference manual covers the basic library. The collection of basic geometric algorithms and datastructures currently includes polygons, boolean operations, convex hulls, triangulations, smallest enclosing circle, and range and segment trees.

Organization of this Manual

This document is organized as follows. Chapter to present successively the following data structures and algorithms: polygon, convex hull, triangulation, smallest enclosing circle, and range and segment trees.



Some functionality of a number of classes and algorithms is considered more advanced, for example because integrity of the underlying datastructure is not guaranteed. In such cases the programmer is responsible for integrity. These functionalities are described in sections like this one, bounded by horizontal brackets.

What is missing in this Release

This release of CGAL provides mostly two-dimensional primitives data structures and algorithms. Higher dimensional versions will be available soon. More geometric datastructures and algorithms will become available in future releases, for example spatial subdivisions.
