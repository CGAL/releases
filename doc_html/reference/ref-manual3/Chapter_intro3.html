Introduction

CGAL, the Computational Geometry Algorithms Library, is written in C++ and consists of three parts. The first part is the kernel, which consists of constant size non-modifiable geometric primitive objects and operations on these objects. The objects are parameterized by a representation class, which specifies the underlying number types used for calculations. The second part is a collection of basic geometric datastructures and algorithms, which are parameterized by traits classes that define the interface between the datastructure or algorithm, and the primitives they use. The third part consists of non-geometric support facilities, such as `circulators', random sources, I/O support for debugging and for interfacing CGAL to various visualization tools. Some of these facilities may depend on other tools or libraries, such as LEDA.

This part of the reference manual covers the support library.

Organization of this Manual

This document is organized as follows. Chapter is about number types that can be used with CGAL. Chapter introduces the notion of circulators, a concept like STL iterators, used for inherently circular sequences. Circulators are used in many algorithms and datastructures of the basic library. Chapter describes a number of generators for random numbers and geometric objects. All classes in the CGAL kernel provide input and output operators for IOStreams; they are described in Chapter . Chapter introduces the notion of color for graphical IO. The window stream, which is derived from Leda windows, is described in Chapter .
